# Contributing Guide

## Branch Stratejisi

- `main`: stabil dal
- `feat/*`: yeni ozellik
- `fix/*`: hata duzeltme
- `chore/*`: altyapi/iyilestirme

## Commit Kurali

Conventional Commits formatini kullan:

- `feat: ...`
- `fix: ...`
- `chore: ...`
- `docs: ...`
- `test: ...`

## Pull Request Kontrol Listesi

- [ ] Testler localde gecti
- [ ] Lint hatasi yok
- [ ] Dokumantasyon guncellendi
- [ ] Geriye donuk uyumluluk kontrol edildi

## Kod Gozden Gecirme

PR aciklamasinda su alanlar zorunlu:

- Problem tanimi
- Cozum yaklasimi
- Etkilenen moduller
- Test kaniti
